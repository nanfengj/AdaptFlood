## 一、项目基本信息

### 1.1 项目目的

**核心目标**：开发一个GPU加速的城市内涝模拟程序

**应用场景**：
- 城市排水管网的出流（点源）→ 地表二维扩散 → 积水分析
- 多个检查井同时溢流，模拟地表积水过程
- 大规模网格（目标：10000×10000网格，10000+点源）

**技术路线**：
- 基于LISFLOOD-FP原版的acceleration模式（惯性波方程）
- 使用Kokkos并行框架实现GPU加速
- 支持NVIDIA CUDA，理论上可移植到国产GPU（海光DCU等）

## 四、技术细节（重要！）

### 4.1 加速模式核心公式

```cpp
// 半隐式动量方程（Bates et al. 2010）
// 原版位置：fp_acc.cpp:76

Q = (q0 - g*dt*hflow*Sf) / (1 + g*dt*hflow*fn*fn*qvect/hflow^(10/3)) * dx

其中：
- q0 = Qxold/Qyold（前一时间步流量，单位m²/s）
- Sf = -dh/dx（水面坡度）
- hflow = max(z0+h0, z1+h1) - max(z0, z1)（流动水深）
- qvect = sqrt(qx² + qy²)（2D速度矢量）
- fn = Manning糙率
- 最后乘dx转换为m³/s
```

### 4.2 数组索引规则（关键！容易出错）

```cpp
// LISFLOOD使用的数组大小和索引：

H[xsz × ysz]           // 水深，索引: i + j*xsz
DEM[xsz × ysz]         // 高程，索引: i + j*xsz

// 【重要】Qx和Qy数组大小都是 (xsz+1) × (ysz+1)
Qx[(xsz+1) × (ysz+1)]  // X方向流量，索引: i + j*(xsz+1)
Qy[(xsz+1) × (ysz+1)]  // Y方向流量，索引: i + j*(xsz+1)
Qxold, Qyold 同上

// UpdateH中的索引（iterateq.cpp:80-82）：
// 对于网格(i,j)：
Qx_in  = Qx[i + j*(xsz+1)]
Qx_out = Qx[(i+1) + j*(xsz+1)]
Qy_in  = Qy[i + j*(xsz+1)]
Qy_out = Qy[i + (j+1)*(xsz+1)]

dV = Tstep * (Qx_in - Qx_out + Qy_in - Qy_out)
H += dV / dA
```

### 4.3 点源坐标转换（容易出错！）

```cpp
// LISFLOOD使用顶部向下的y坐标系统
// 原版位置：input.cpp:1397-1398

double tly = yll + nrows * cellsize;  // 顶部y坐标
ps.x = (int)((x_world - xll) / cellsize);
ps.y = (int)((tly - y_world) / cellsize);  // 注意：tly - y，不是 y - yll
```

### 4.4 点源流量公式

```cpp
// 原版位置：iterateq.cpp:71
// q是单位宽度流量(m²/s)，需要乘dx得到体积流量

H[idx] += q * dx * Tstep / dA;

// 等价于：
// 体积增量 = q(m²/s) × dx(m) × dt(s) = m³
// 水深增量 = 体积 / 面积 = m³ / m² = m
```

### 4.5 重要参数设置

```
# test.par 关键参数

initial_tstep   2.0     # 初始时间步，不要设太大（≤2秒），否则数值不稳定！
cfl             0.7     # CFL系数
fpfric          0.035   # Manning糙率
```

## 五、曾经遇到的问题和解决方案

### 5.1 问题：水量只有CPU的1/10

**原因**：点源公式缺少dx乘数
**解决**：`H += q * dx * dt / dA`（原来少了dx）

### 5.2 问题：点源位置相反

**原因**：y坐标使用底部向上索引
**解决**：改为顶部向下：`y = (tly - y_world) / dy`

### 5.3 问题：水不扩散，集中在点源

**原因**：使用的是扩散波方程（无动量），不是加速模式
**解决**：实现完整的加速模式（半隐式动量方程 + Qold数组）

### 5.4 问题：体积暴增3倍

**原因**：Qy数组大小错误（xsz × (ysz+1) 应该是 (xsz+1) × (ysz+1)）
**解决**：统一Qx和Qy数组大小为 (xsz+1) × (ysz+1)

### 5.5 问题：initial_tstep=10导致不稳定

**原因**：第一步时间步太大，产生数值振荡
**解决**：initial_tstep改为2.0
